## 为何要使用这类东西？

主要是为了更“全局”地部署代理。

以便于更方便地进行学术研究学习、计算机程序开发资源获取。

以往是直接在计算机操作系统内部使用代理软件（SS/SSR/V2rayN）部署代理，然而这种方式有很多局限性。

因为虽说这几种代理软件都具有全局模式，但这种模式对于某些场景是无效的。比如在本地 Git 与 GitHub 通信（push / pull / ……）时，假如不对本地 Git 作特殊的系统配置，而是直接在代理软件开启“全局模式”，结果是无效的。
